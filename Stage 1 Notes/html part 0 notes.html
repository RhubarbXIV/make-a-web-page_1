Lecture 1

The Basics of the web

HTML, or hypertext markup language is the basic document type of the itnernet. When a computer browser requests a web page, a HTTP request is sent to a server, which typically fulfills said request by providing the corresponding HTML file, which is then in turn interpreted and displayed by a browser. More information can be found here.

HTML introduction

So far in class we've discussed how HTML can be used to display text, images and links. Basic HTML is a good introduction to computing because it allows for an easy comparison between input and output, e.g. the difference between the markup language used and the interpreted result from the browser. It is commonly said that "computers are stupid," but I prefer to think of them as being hopelessly literal- only the most explicit well-formed instructions are going to have the desired effect.

HTML elements and tags

The basic unit of HTML markup is called an element, which can represent a simple line break tag <br>, or a link to a site with a specific title: <a href="url">"title"<a/>. In this case, the tag is the content within the brackets, "url" is a value.

Some elements are inline, and effect the text in just a specific place, while others are block elements, which can be said to form an invisible box around their content for manipulation. A line break is an example of the former, while a paragraph element is an example of the latter.

Lecture 2

Webpages, Documents, and Structure

Webpages are, on a high level, trees of boxes. Within the Document Object Model (DOM), HTML represents the "what" to display, and CSS (cascading style sheets) represents the "how"

Conceptualizing webpages as trees of boxes seems to make liberal use of the <div> tag, often embedded within other <div> tags. This seems especially helpful as a starting point where boxes are first sketched out and then populated with the specific elements of the desired page.

Lecture 3

Introduction to CSS

Style information about a HTML document is stored separately in a CSS file. The style file must be linked to the HTML file to be effective (use link tag)

CSS applies to HTML elements with trivial specificity, exhibiting inheritance. Put another way, embedded elements are said to inherit style information from the parent element, which in turn inherits its style information from its parent and so on and so on.

The Box Model

All HTML elements exist as theorteical "boxes" with margin, border, padding and content. All these attributes and more can be altered with instructions from a corresponding CSS document.

One of the most complicated concepts in CSS is how these boxes are to be positioned. A powerful tool for managing box positioning is flexbox (click here for more information). This looks like a topic with lots of room for further exploration

Avoiding Repetition

One of the most powerful applications of CSS, and by extension computer programming in general, is that it allows one to avoid repetitive instructions. This greatly benefits the designer of code, saving development time in writing out instructions and perhaps equally importantly, limiting the chances of making an error that causes a program to behave in an unexpected fashion.

Naming classes in <div> elements, and then supplementing that class with style information from a corresponding CSS document is one example of how a developer might avoid repetition. All elements of the same class can be altered at once with a change to the style instructions contained within the CSS.
